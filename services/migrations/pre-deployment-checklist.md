# 生产环境部署前检查清单

本文档提供生产环境数据库迁移的完整检查清单，确保部署过程安全可控。

---

## 部署前准备（T-24小时）

### 1. 迁移脚本审查

- [ ] 已在开发环境测试迁移脚本
- [ ] 已在测试环境验证迁移脚本
- [ ] 迁移脚本不包含 `DROP DATABASE` 语句
- [ ] 迁移脚本不包含 `TRUNCATE TABLE` 语句
- [ ] 迁移脚本不包含删除生产数据的操作
- [ ] 所有 SQL 语句都经过代码审查
- [ ] 迁移脚本包含验证步骤
- [ ] 迁移脚本使用幂等性设计（可重复执行）

### 2. 回滚计划准备

- [ ] 已创建对应的回滚脚本
- [ ] 回滚脚本已在测试环境验证
- [ ] 回滚脚本命名正确（`*.rollback.sql`）
- [ ] 回滚步骤已文档化
- [ ] 团队成员了解回滚流程

### 3. 数据备份计划

- [ ] 确认备份存储位置有足够空间
- [ ] 备份脚本已准备就绪
- [ ] 备份验证方法已确定
- [ ] 备份保留策略已明确
- [ ] 备份恢复流程已测试

### 4. 性能影响评估

- [ ] 评估迁移对数据库性能的影响
- [ ] 确定是否需要锁表及锁表时间
- [ ] 评估索引创建时间
- [ ] 确定最佳执行时间窗口（低峰期）
- [ ] 准备性能监控方案

### 5. 团队协调

- [ ] 通知相关团队成员部署计划
- [ ] 确认部署时间窗口
- [ ] 分配角色和责任
  - 执行人：___________
  - 监控人：___________
  - 备份人：___________
  - 回滚决策人：___________
- [ ] 建立沟通渠道（如：群聊、电话）
- [ ] 准备应急联系方式

## 回滚流程（如需要）

### 回滚决策标准

满足以下任一条件时应考虑回滚：

- [ ] 迁移执行失败且无法修复
- [ ] 数据完整性验证失败
- [ ] 应用核心功能异常
- [ ] 性能严重下降（超过预期 50%）
- [ ] 出现数据丢失或损坏
- [ ] 用户无法正常使用服务

### 回滚执行步骤

1. **停止应用服务**
   - [ ] 停止所有应用实例
   - [ ] 通知用户系统维护中

2. **执行回滚脚本**
   - [ ] 记录回滚开始时间：___________
   - [ ] 执行回滚脚本
     ```bash
     mysql -u username -p database_name < migrations/YYYYMMDD_HHMMSS_description.rollback.sql
     ```
   - [ ] 记录回滚结束时间：___________

3. **验证回滚结果**
   - [ ] 验证数据库架构已恢复
   - [ ] 验证数据完整性
   - [ ] 检查数据库错误日志

4. **恢复应用服务**
   - [ ] 重启应用服务
   - [ ] 验证应用功能正常
   - [ ] 通知用户服务已恢复

5. **问题分析**
   - [ ] 分析回滚原因
   - [ ] 记录问题详情
   - [ ] 制定修复计划
   - [ ] 更新 `migration-history.md`

## 附录：常用命令

### 数据库备份

```bash
# 完整备份
mysqldump -u username -p database_name > backup.sql

# 仅备份架构
mysqldump -u username -p --no-data database_name > schema.sql

# 仅备份数据
mysqldump -u username -p --no-create-info database_name > data.sql

# 压缩备份
mysqldump -u username -p database_name | gzip > backup.sql.gz
```

### 数据库恢复

```bash
# 从备份恢复
mysql -u username -p database_name < backup.sql

# 从压缩备份恢复
gunzip < backup.sql.gz | mysql -u username -p database_name
```

### 数据库监控

```sql
-- 查看当前连接数
SHOW STATUS LIKE 'Threads_connected';

-- 查看最大连接数
SHOW VARIABLES LIKE 'max_connections';

-- 查看慢查询
SHOW STATUS LIKE 'Slow_queries';

-- 查看表大小
SELECT
  TABLE_NAME,
  ROUND(DATA_LENGTH/1024/1024, 2) AS 'Data_MB',
  ROUND(INDEX_LENGTH/1024/1024, 2) AS 'Index_MB',
  ROUND((DATA_LENGTH + INDEX_LENGTH)/1024/1024, 2) AS 'Total_MB'
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_SCHEMA = 'database_name'
ORDER BY (DATA_LENGTH + INDEX_LENGTH) DESC;

-- 查看正在执行的查询
SHOW PROCESSLIST;
```
