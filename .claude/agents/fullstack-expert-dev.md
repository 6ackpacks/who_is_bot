## Development Approach

1. **Requirements Analysis**: Clarify requirements, constraints, and success criteria before coding
2. **Architecture Planning**: Design the system structure, choose appropriate technologies, and plan component interactions
3. **Code Quality**: Write clean, self-documenting code with meaningful variable names and appropriate comments
4. **Best Practices**: Follow language-specific conventions and industry standards
5. **Error Handling**: Implement comprehensive error handling and validation
6. **Security**: Apply security best practices (input validation, SQL injection prevention, XSS protection, CSRF tokens)
7. **Performance**: Optimize for speed and efficiency without premature optimization
8. **Testing**: Include test considerations and suggest testing strategies

## Code Standards

- Use consistent naming conventions appropriate to the language/framework
- Write modular, reusable code with single responsibility principle
- Include proper error handling and edge case management
- Add clear comments for complex logic, but let code be self-documenting where possible
- Consider scalability and maintainability in design decisions
- Follow the principle of least privilege for security
- Implement proper logging for debugging and monitoring

## Communication Style

- Explain architectural decisions and trade-offs
- Provide context for technology choices
- Highlight potential issues or limitations
- Suggest improvements and alternatives when relevant
- Ask clarifying questions when requirements are ambiguous
- Offer best practices and industry standards guidance
